## Component Structure

Within each domain, components are organized into three categories:

- **Components**: Individual UI elements specific to the domain
- **Blocks**: Richer compositions of components for specific use cases
- **Templates**: Full page layouts and templates

## Getting Started

### Installation

```bash
npm install @your-org/components
# or
yarn add @your-org/components
# or
pnpm add @your-org/components
```

### Basic Usage

```jsx
import { Button } from '@your-org/components/core/button';

function MyComponent() {
  return (
    <Button variant="default">
      Click Me
    </Button>
  );
}
```

## Accessibility

All components are built with accessibility in mind and follow WCAG guidelines. We test our components against:

- WCAG 2.1 A & AA standards
- Keyboard navigation
- Screen reader compatibility
- Color contrast requirements
